# 课程定位

AI × Web3 School 是一个为期 4 周的线上共学营 + Hackathon，目标是帮助开发者：

1. **建立 AI 与 Web3 的交叉认知**：理解 LLM、Agent、工作流如何与区块链账户、合约、交易结合
2. **完成最小可行实践**：至少跑通一次 AI 工具调用、一次测试网交互、一个交叉实验
3. **产出可展示的作品**：通过 Hackathon 完成一个方向明确的 demo，并积累 proof-of-work
4. **形成持续学习习惯**：学会用 learning agent 辅助学习、任务拆解、日志记录和复盘

### 面向谁？

- 具有 Web3 背景的参与者 ：希望系统地了解 LLM、代理、工作流程、AI 编码、工具使用以及 AI 系统的边界的构建者。
- 具有人工智能背景的参与者 ：希望学习 Web3 基础知识（如账户、钱包、gas、智能合约、权限、安全执行和链上验证）的构建者。
- 开发人员、研究人员、技术项目经理、开发者关系人员和生态系统运营人员 ：希望识别具体的 AI × Web3 问题并将其转化为项目方向的人员。
- 高潜力学生或早期建设者 ：参与者可能尚未达到高年级水平，但愿意进行建设、记录过程，并将想法推向可验证的结果。

### 你最终会获得什么？

- 用通俗易懂的语言来理解 AI × Web3，超越那些流行术语。
- 从基础学习和方向选择到实践操作和项目提交的完整过程。
- 一个可以展示、审查和继续推进的项目成果。
- 直接体验过 AI 执行能力、Web3 权限边界、链上验证、安全性和隐私性、治理协调以及构建的真正困难。
- 通过 WCB 的展示/机会网络，获得后续展示、联系和机会匹配的途径。

# 课程大纲

## （1）第一周 | 人工智能和 Web3 基础知识

通过将 LLM（生命周期管理）、提示、工作流、代理、工具使用、AI 编码、钱包、签名、交易、gas、智能合约和测试网连接成一条执行链，构建一种共享语言。参与者将完成 Vibe Coding/Hermes Agent 的练习，创建测试钱包，进行测试网交易，部署或调用一个最小智能合约，并记录成功、失败和人工修正。

## （2）第二周 ｜ 人工智能 × Web3 交叉领域

通过诸如智能商务/支付、开发工具、人工智能安全/隐私、人工智能×治理/协调以及开放方向等主题，深入探讨实际问题。参与者选择一个方向，规划任务发起人、执行人、付款人、验证人以及风险承担人，然后制定一份提案，进入下一阶段。

## （3）第三周 | 练习深化和黑客马拉松启动

参与者继续完善细节，完成一些小型练习，例如代理工作流程+钱包确认、链上收据、测试网支付、治理提案摘要、风险控制策略或可验证的执行记录。与此同时，各团队最终确定黑客马拉松的路线、项目主题、目标用户、技术路径、角色和演示计划。

## （4） 第 4 周 | 黑客马拉松冲刺和演示展示

团队的工作重点是核心实现、基础测试、README 文件、演示流程以及必要的链上或工具调用记录，并通过导师的答疑时间获得反馈。最终提交的内容包括项目名称、方向、目标用户、问题定义、演示链接、代码仓库、README 文件、视频以及相关的合约地址、测试网地址、交易哈希或其他验证材料。

# 挑战任务

learning agent 配置、GitHub repo、AI coding、Web3 测试网、最小交叉实验、方向扫描、Hackathon proposal。

# 🔒 安全边界规范

> ️ **重要提示**：以下内容涉及安全操作，必须严格遵守。

## 绝不允许交给 Agent 的信息

以下信息**必须人工保管**，**绝不允许**写入任何配置文件、代码、日志或与 agent 的对话中：

- ❌ **私钥（Private Key）**：钱包的私钥、签名密钥
- ❌ **助记词（Mnemonic / Seed Phrase）**：12/24 个单词的恢复短语
- ❌ **API Key / Secret**：交易所、Infura、Alchemy、OpenAI 等服务的密钥
- ❌ **密码（Password）**：钱包密码、账户密码、加密文件的密码
- ❌ **签名请求（Signing Request）**：未经人工确认的签名操作

## 必须人工确认的操作

以下操作**必须人工确认**后才能执行，不允许直接交给 agent：

1. **转账操作**：任何涉及资产转移的交易，必须人工核对地址、金额、Gas
2. **合约部署**：部署合约前必须人工审查代码，确认无后门、无漏洞
3. **签名授权**：授权合约访问代币或 NFT 前，必须确认授权范围和期限
4. **权限变更**：修改多签阈值、添加/移除管理员等敏感操作
5. **生产环境部署**：从测试网切换到主网前，必须经过充分测试和审计

## Agent 可以协助的操作

- ✅ 生成代码模板、解释代码、debug
- ✅ 查询公开文档、总结文章、对比方案
- ✅ 规划学习任务、拆解步骤、提醒进度
- ✅ 生成测试用例、模拟数据、示例交易参数
- ✅ 分析交易日志、解读错误信息、提供修复建议

## Human-in-the-Loop 最佳实践

- 使用 `.env` 文件存储敏感信息，并加入 `.gitignore`
- 使用环境变量而非硬编码密钥
- 定期轮换 API Key，撤销不再使用的密钥
- 使用硬件钱包存储大额资产
- 对重要操作启用多重确认（邮箱、短信、2FA）
- 保留操作日志，便于追溯和审计

# 🤖 Hermes Agent 交互式学习计划

> **使用说明：** 学员配置好 Hermes Agent 之后，直接把本文件的 URL 发给 agent，让它理解共学营目标、课程大纲、挑战、关键时间点和个人学习计划。

## 启动流程

当学员首次使用 Hermes Agent 时，agent 应按以下顺序进行交互式追问：

### Step 1：当前水平评估

```
📝 请评估你的当前水平：

- [ ] 初级：刚入门编程，对 AI 和 Web3 都了解不多
- [ ] 中级：已有工程经验，但对 AI 或 Web3 不熟悉
- [ ] 高级：在 AI 或 Web3 某一领域已有积累
```

### Step 2：背景调查

```
🎯 你的技术背景更偏向哪一边？

- [ ] 更偏 AI：做过机器学习、NLP、大模型应用
- [ ] 更偏 Web3：写过合约、用过 DeFi、参与过 DAO
- [ ] 两边都刚入门：都是新手，想同时学习
- [ ] 已有工程经验：后端/前端/全栈，想拓展到新领域
```

### Step 3：时间与目标

```
⏰ 你每周可投入多少时间？

- [ ] 5-10 小时：只能利用碎片时间
- [ ] 10-20 小时：有固定的学习时间
- [ ] 20+ 小时：可以全职或接近全职投入

🎯 你的主要目标是什么？

- [ ] 补基础：系统学习 AI 和 Web3 基础知识
- [ ] 做 demo：尽快做出一个可展示的作品
- [ ] 研究方向：探索某个交叉方向的深度问题
- [ ] 准备 Hackathon：以获奖或孵化为目标
```

### Step 4：生成个人计划

根据以上回答，Hermes Agent 应生成：

#### 1. Week 1-2 个性化任务计划

- **每日学习任务**（具体到章节、视频、练习）
- **预期完成时间**
- **验证标准**（如何判断自己掌握了）

#### 2. Repo 结构建议

```
AI-Web3-School-note/
── week1/
│   ├── ai-basics/        # AI 基础笔记和代码
│   ├── web3-basics/      # Web3 基础笔记和代码
│   ├── cross-experiment/ # 交叉实验记录
│   └── journal.md        # 学习日志
├── week2/
│   ├── direction-research/ # 方向调研笔记
│   └── journal.md
├── week3/
│   ├── hackathon-proposal/ # Proposal 文档
│   └── project-repo/       # 项目代码（可单独建 repo）
├── week4/
│   ├── demo/               # Demo 材料
│   └── retrospective.md    # 复盘笔记
── README.md
```

#### 3. Proof-of-Work 清单

- 每日/每周需要提交的验证材料
- 格式要求（截图、哈希、代码片段等）
- 提交方式和截止时间

### Step 5：持续跟进与复盘

每次学习结束后，Hermes Agent 应主动追问：

```
 今天的学习复盘：

1. ✅ 今天完成了什么？（列出具体任务）
2. ❓ 遇到什么问题？（技术难点、概念困惑、时间管理）
3.  下一步需要补什么材料？（推荐相关阅读或练习）
4. 🎯 明天计划做什么？（调整后的计划）
```

## Agent 监督机制

Hermes Agent 应根据以下规则监督学习进度：

### 每日检查

- 提醒学员当天的 Co-Learning 时段和讲座安排
- 检查是否完成了每日学习任务
- 询问学习进展和遇到的问题

### 每周检查

- 确认本周挑战任务的完成进度
- 检查 proof-of-work 材料的完整性
- 评估是否按时提交 Week 1 任务

### 里程碑检查

- 在 Week 1 结束前提醒提交 proof-of-work
- 在 Week 2 结束前确认方向选择和团队组建
- 在 Week 3 中期提醒提交 Proposal
- 在 Week 4 结束前确认 Demo 准备情况

### 安全提醒

- 在进行 Web3 相关操作前，提醒安全边界规范
- 在进行合约部署或转账前，要求人工确认
- 提醒使用 `.env` 文件存储敏感信息

# ❓ 常见问题

### Q1：我没有编程经验，能参加吗？

A：可以，但需要投入更多时间补基础。建议优先学习 Python 或 JavaScript 基础，再进入 AI 和 Web3 内容。

### Q2：我需要准备多少钱？

A：全程使用测试网，不需要真实资金。只需准备少量 ETH 用于测试网 Gas（可从水龙头免费获取）。

### Q3：我没有 GPU，能跑大模型吗？

A：不需要。课程使用 MaaS（Model-as-a-Service）API，只需 API Key 即可调用顶级模型，按 token 付费。

### Q4：我可以中途加入吗？

A：可以，但建议从 Week 1 的内容开始补起。

### Q5：我一个人没有队友，怎么办？

A：可以在社群中寻找队友，也可以单人参赛。

### Q6：Hermes Agent 能帮我写代码吗？

A：可以，但要注意安全边界。所有涉及资产和权限的操作必须人工确认。

### Q7：如何验证我的学习成果？

A：通过 proof-of-work 材料：GitHub commit 记录、交易哈希、合约地址、截图、学习日志等。
